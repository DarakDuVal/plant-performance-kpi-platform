# Architecture Decision Records (ADRs)

This directory is a **template location for your own project's decisions**, not a log of decisions about this agent
catalog. When the agents work on your software project, they record its significant architecture and design decisions
here.

## How to use

- Copy [`ADR-000-template.md`](ADR-000-template.md) for each new decision.
- Number ADRs sequentially: `ADR-001`, `ADR-002`, …
- Record a decision when it affects multiple components, has non-obvious trade-offs, or would be costly to reverse.
- ADRs are immutable once accepted. A superseded decision gets the status `Superseded by ADR-NNN` instead of being
  deleted.

See [`../quality-standards.md`](../quality-standards.md) for the full ADR policy.

This catalog intentionally keeps **no ADRs of its own** — the only file here besides this README is the template, so
your project starts from a clean slate.
